# Naming and Tags（命名和标签）

## 概述

标签用于明确布局的导入方式，或将脚本分配给 Unity 中的某些对象。

**组件命名格式**：`"tag - name - info"`

> **注意**：大多数组件不需要标签。

## 命名规则

### 1. tag（标签）

- **作用**：组件标签，导入器需要依靠它来理解如何准确导入该对象
- **位置**：标签是组件名称的一部分

### 2. 分隔符

- **符号**：`-` 或 `/`
- **作用**：必需的分隔符，将标签与后面的内容分开

### 3. name（名称）

- **作用**：组件名称
- **说明**：在此指定组件的逻辑用途

**示例**：`btn - menu open`

### 4. info（信息）

- **作用**：您想在此组件名称中包含的任何信息

## 命名示例

```
btn - menu open
bg - circle pattern
img - avatar
```

## 标签兼容性处理

如果遇到标签在新版本资源中已被重命名并出现兼容性问题，您可以要求开发人员在 **FcuConfig** 文件的 **Tags** 数组中的 **"Figma Tag"** 字段更改值。
