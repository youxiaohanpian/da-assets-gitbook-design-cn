> **提示：** 标签用于明确布局的导入方式，或在 Unity 中为某些对象分配脚本。
> 带标签的组件名称格式：`"标签 - 名称 - 信息"`。
> 大多数组件不需要标签。

1. **"标签"** — 组件的标签。导入器需要它通过标签来准确理解如何导入此对象。标签是组件名称的一部分。

2. **"-" 或 "/"** — 必需的分隔符。将标签与其后的所有内容分隔开。

3. **"名称"** — 组件名称。在此指定组件的逻辑用途，例如：
   - `btn - 菜单打开`

4. **"信息"** — 你希望包含在此组件名称中的任何信息。

5. 使用 "-" 分隔符的示例：
   - `btn - 菜单打开`
   - `bg - 圆形图案`
   - `img - 头像`

6. 如果标签在新版本资产中已被重命名，并且你遇到兼容性问题，可以要求开发者在 `FcuConfig` 文件中更改 `Tags` 数组中的 `Figma Tag` 字段的值。
