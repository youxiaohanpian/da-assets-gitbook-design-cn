# 组件的唯一性

**为防止**同名的组件在下载后相互**覆盖**，在下载前会根据这些组件及其内部所有组件的属性，向它们的名称添加一组随机数字。  
如果这些属性发生变化，数字集合也会变化，然后就会下载一个新文件。

为避免项目中的预制体和图像重复，请在 Figma 中使用组件创建功能。

从 2.1.0 版本开始，你可以给组件**相同名称**，如果这些组件及其子组件的属性不同，它们将**不会相互覆盖**。

> **警告：** 如果在导入后，你的项目遇到以下问题，请联系开发者：
> 1. 精灵（Sprites）在你认为不应该重复时却被重复了；
> 2. 应该是不同精灵的图像最终共享了同一个精灵。
