布局规则

提示： 前两点是强制的。 在大多数情况下，你不需要在导入前对布局做任何其他修改，就能获得正确的结果。

  1. 导入前，禁用 Frame 上的 Clip Content（裁剪内容）。

  2. 如果要导入 Section 内容，需要将其转换为 Frame：

    • 右键点击 Section
    • 在出现的上下文菜单中，选择 “Convert to Frame”（转换为 Frame）

  3. 如果你希望某个组件被导入为单个图像，但默认情况下没有发生，请参考 图像合并 章节。

  4. 如果你希望组件内的子组件不被合并为单个图像，请参考 图像分割 章节。

  5. 标签后的文本不应仅由特殊字符组成。

  6. 包含子文本组件或任何其他阻止父组件被合并为单个 PNG 图像的元素的父组件规则：

    • 不要在父组件上放置 **Effects（效果）**和 Image-Fill（图像填充）
    • 如果你使用的是 UI.Image 或 SpriteRenderer，不要在父组件上放置渐变（Gradients）
    • 如果你使用的是 SpriteRenderer，不要给父组件添加圆角

    如果违反了这些条件，此类父组件的填充和效果将不会被导入。要解决此问题，请将这些填充和效果移动到单独的子组件中。

  7. Unity 提供了多种工具来复现 Figma 的图形能力，例如支持渐变和描边的程序化图像。请查看 Figma 组件与 Unity 组件兼容性表，以更好地了解导入后的设计效果。

  8. 请查看 示例模板，了解在 Figma 中设计布局以供导入时需要注意什么。
